floviz

Project Website

Team Members: Jonah Sussman, Christian Holcombe, Andy Hansen, and Sam Perry

The goal of the project is to create an app that makes water data easily manipulated and more visually appealing to study and display. Many existing apps focus on weather events and displaying daily data, but our app will focus on trend visualization and be used mainly for research and educational purposes. This data will be sourced from both the client, Alabama Water Institute, and from publicly available APIs.

This project will be considered a success if we can display water data in a way that is easy to understand, coherent, and teaches the users more about the environment. The interface for the app will be a map with various overlays and data points, and allow users to select which data they would like to view. The app will focus on the state of Alabama to begin, and, if the project timeline allows, expand to the entire continental US.

Project Goals

The main goal of our project is to provide a user-friendly web application that can be used to visualize trends and data uploaded to the server.

More Specific Goals Include:

  1. Setting up a web server to host our application.
  2. Creating a dynamic front end to request data.
  3. Reading data from multiple services to conglomerate water data.
  4. Using Geoserver to format and hold data layers.
  5. Tutorial for uploading data layers to Geoserver.

Approach

Tools

floviz is a web-hosted React project. We use Geoserver to handle the layering and storage of data for the front end to pull and display. Additionally, we use Github to upload code and communicate changes to the codebase. The server currently hosting the application is a personal server and will be transitioned to a server maintained by the Alabama Water Institute. For data fetching and scraping, we have used Python and a library, Climata, which scrapes data from specified Alabama streams and creates dynamic visuals for educational purposes.

User-Experience

The user will be presented with a blank map with several different sidebar sections. To the left, the user will be able to select from layers of data currently in Geoserver to be displayed over the map. To the right, there is a begin data and end date which will allow the user to specifiy the dates of the data they want to select. There is also a set of buttons that allow the user to select the interval of time to fetch and load the data as well as select the playback speed of the video returned. Finally, the user can fetch and load data using the fetch data button as well as set the bounding box for the data to be displayed. There also exists buttons at the bottom which have information about our application, displays our github repository, and a link to contact us via email.
