The Water App - Project Site

Team Members: Jonah Sussman, Christian Holcombe, Andy Hansen, and Sam Perry

Although there exists copious amounts of data covering every minute detail about the water, an easily readable and maneuverable app to visualize, study, and learn about this water data does not exist. The goal of The Water App is to provide a coherent application that fills our client's needs, bringing important data about water to the public.

Project Goals

The main goal of our project is to provide a user-friendly web application that highlights water-related metrics provided to us by the Alabama Water Institute and National Weather Service.

More Specific Goals Include:

  1. Setting up a web server to host our application.
  2. Creating a dynamic map interface.
  3. Reading data from multiple services to conglomerate water data.
  4. Creating clean and informative overlays for the data.
Possible Goals:
  1. Creating user authentication for web app.
  2. Setting up database to store user information.

Approach

Tools

The Water App is a REACT web-hosted project. It is currently running on personal servers (DigitalOcean) for development and testing, but will be moved later to find a more permanent solution. It uses the Geoapify API for the creation of the dynamic map that will display our data. Leaflet is a JavaScript open-source library that allows simple and readable overlays to be created to visualize our data. The data will come from the Alabama Water Institute and the National Weather Service.

User-Experience

The app is aiming to be an app that all users regardless of knowledge level. When a user first visits our website, they will be shown a plain map of the United States, focused on Alabama. There will be several overlay options that the user can click on and show the water data they want on the map. There will be a location search bar for different US cities, states, and coordinates, but the user can also drag the map around to visit different locations. In the future, the user may be able to log in and be shown their previous search instead of a plain map. The user may be able to click on specific points in the data and get in depth information about the specific water metric.
